Python Projects

Natural Language Processing

Application of various NLP-Consepts to mainly Swiss German text. The data was partly written manually by myself and partly collected by parsing / processing publicly available content.

The most important Techniques that were applied are: Classification of Text input to the relevant Intent (using a Neural Network with the Bag of Word Model and various n-grams); Converting text-written Numbers to Integers (using an own written algorithm to consider several variations, namely to handle various dialects); working with Part of Speech (POS) Tags (using a customised Hidden Markov Model).







Translator for Swiss German

There are many different usecases for this this processed and classified data, one of them being a multi-directional translator. It is mainly made for Swiss and Standard German, but can easily be used for any other Language, with an additional step.

Besides a simple one-word translation from Swiss German to Standard German and vice versa, I am implementing a Version that allows full sentence translation. Using the POS Classification and Markov Chains, a correct grammatical structure is maintained for the translation output.





Autocompletion Tool

Another application of the processed data is an autocompletion, including the option to predict upcomming words.

I specifically implemented this for the Swiss German Language, mainly using n-grams, as well as a Model based on a Markov Chain. It is built in a way, that it can easily be adapted to any other language, by simply providing plain texts in the desired language to the model.





Virtual Assistant

In a first version it was an english voice assistant, incl. Speech Recognition as well as Text to Speech responses. However, I later focused on Swiss German, for which I (mainly due to the lack of voice data) am currently only working with text-input and text (or optional Standard German Speech) output.

Some of the featureas are Navigation, Math and information about the Weather It can also answer general questions about things like Countries, People, Dates (incl. parsing of numbers in text format) or dictionary related questions.





Text Recognition "Snipping Tool"

Provides a Keyboard Shortcut to chose a part of their screen to perform an OCR (Optical Character Recognition). The text in this area is extracted and whitespaces are handelet, all using a pretrained Neural Network, leading to offline capabilities.

The text is automatically copied to the Clipboard. Whit an internet connection it can be processed further: For example automatically translated to any other language or used for an online research. This research feature can especially be usefull to automatically provide links to the full source (e.g. a Blog or GitHub Repository) of any input (e.g. a Code Snippet).





Sudoku with Animated Solver

This is the well known logic-based Puzzle Game called Sudoku. There is a Version available with a PyGame GUI and one Version is playable in the Terminal, using no 3rd party libraries. It starts with some given numbers and the player needs to fill in all empty fields following this simple rule: Any number can only occure once in every line, column and separated 3x3 gird.


Comes with an Artificial Intelligence using the Backtracking Algorithm. It includes animated solving (exact Backtracking approach, but slowed down for illustrative purposes) as well as instant feedback after a move or for hints if required.





Tic Tac Toe with AI

Well known Tic Tac Toe Game, incl. and AI Player using the optimized MiniMax Algorithm (incl. Alpha / Beta Pruning).

Including features like live evaluation of the game and suggesting better moves. This might be developped further (e.g. better animated) and adapted to other games like Chess, which is also solved using the MiniMax Algorithm.

Web-Dev Projects

easyBGE Website

A Website to view the Verdicts of the Swiss Federal Court in a more user friendly way and additional features like a remembered Dark Mode (using cookies & localstorage) and a Chrome Extension.

First of all, footnotes are inserted instead of the inconvenient long comments in parantheses. Then links and previews (shown on hovering; implemented using my own API) for legislation are generated. Finally it the old website is converted to a responsive design, to make mobile use far more convenient.





Legislation API

For the easyBGE Website I wanted to implement a preview on hovering over legislation. There was currently no way to get this data, therefore I bult my own API for this purpose (?==> iwie, dass sie auch öffenltlich verfügbar sei, wohl für v1 Website noch nicht...).

The functionality is pretty straight forward: The user can just use GET-Requests, providing the desired Law and Article (optionally also the Section / Subsection) and will get the data back in JSON format.





Chrome Extension

Many users might wanna continue to search the verdicts on Google and therefore reach the outdated website of the Swiss Federal Court, insted of my improved Version. Therefore I also decided to develop a chrome extension with the same purpose.

First of all it offers the option, to simply redirect to my website upon visiting a verdict on the official site. Otherwise it provides the core functionality on the same page using DOM-Manipulation. Furthermore I planned some space to potentially use for ads for my Website or third party Customers.






Finance Application with Login and SQL

Web Application, that allows user to Log In, look up Stock Prices and make transactions. Live data achived by using the Real Time Prices provided by the 3rd party IEX Financial API.

Includes further features like a Responsive Design, full transaction History (using a SQL Database) and the option to add more Money to the account.
